## 2024-05-22 - Persistent Re-renders in List Components
**Learning:** `React.memo` combined with `framer-motion` layout animations and `zustand` subscriptions can be tricky to optimize. In `GameCard.tsx`, even with `React.memo` and strictly equal props, re-renders were observed during state updates. This suggests that either `framer-motion` propagates context changes that bypass `React.memo`, or `zustand` selectors, while preventing value updates, still trigger component re-evaluations in some environments (like Strict Mode or specific dev builds).
**Action:** When optimizing lists with `framer-motion` and `zustand`, rely primarily on granular selectors (boolean return values) to minimize the *cost* of re-renders, even if the re-render itself (function call) cannot be fully eliminated. Always verify with production builds if possible, as dev tools/HMR can introduce false positives in re-render counts.
