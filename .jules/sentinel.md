## 2025-12-23 - Missing Iframe Sandbox on Game Player
**Vulnerability:** The `iframe` element in `app/GamePlayer.tsx` used to embed games was missing the `sandbox` attribute. This meant that embedded games (even if local) ran with full privileges, potentially allowing them to access the parent window's DOM, navigate the top frame, or execute plugins if they were compromised or malicious.
**Learning:** Even when embedding "trusted" local content, failing to use `sandbox` violates the principle of least privilege. A missing `sandbox` attribute is a common oversight that leaves the application vulnerable to cross-site scripting (XSS) or hijacking if the embedded content is ever compromised.
**Prevention:** Always apply the `sandbox` attribute to `iframe` elements. Start with the most restrictive set of permissions and only add what is strictly necessary (e.g., `allow-scripts`, `allow-same-origin` for local assets/storage, `allow-pointer-lock` for games). Explicitly omitting `allow-top-navigation` prevents the iframe from redirecting the user.
