## 2025-05-23 - [Missing Sandbox Attribute on Game Iframes]
**Vulnerability:** The application loads games via `iframe` in `GamePlayer.tsx` without the `sandbox` attribute. While games are currently served from the same origin (`/games/`), an unsandboxed iframe has full access to the parent window's DOM, `localStorage`, and `cookies`. If a game were compromised or malicious, it could steal the user's persisted state (auth tokens, achievements) or redirect the user.
**Learning:** Even when content is "first-party" (served from same origin), treating it as untrusted via sandboxing provides defense-in-depth. This is critical for "platform" type apps that host multiple independent sub-applications or games, as it isolates failures and security issues.
**Prevention:** Always apply the `sandbox` attribute to iframes hosting content that does not *need* full access to the parent context. Use the minimum set of permissions required: `allow-scripts`, `allow-same-origin`, `allow-forms`, `allow-popups`, `allow-pointer-lock`.
